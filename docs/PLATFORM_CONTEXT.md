# Контекст платформи

Цей файл потрібен як короткий product context для маркетингових сторінок і лендингів.
Повний технічний source of truth знаходиться у sibling repo `insomnia.workshop`.

## Що це за продукт

Платформа дає змогу запускати й вести Telegram-магазин без окремої кастомної розробки під кожного продавця.

Ядро продукту:
- Telegram-бот для клієнтського сценарію покупки
- Telegram WebApp для каталогу та checkout
- WebApp-хаб для керування магазином
- backend платформи для multi-tenant магазинів

## Для кого продукт

Основна аудиторія:
- власники магазинів, які продають через Telegram
- невеликі бренди та локальні продавці
- магазини, яким потрібен швидкий запуск без окремого сайту або важкої адмінки

## Що вже є в продукті

Перевірені напрямки функціоналу:
- каталог товарів
- кошик і оформлення замовлення
- ручні та автоматизовані сценарії оплат
- накладений платіж
- Нова пошта та робота з ТТН
- WebApp-керування магазином
- тексти, кнопки, модулі й налаштування магазину
- клієнти, кампанії, аналітика, промокоди, крос-продажі залежно від тарифу
- підписка, рахунки, історія оплат і режими доступу магазину

## Як правильно позиціонувати продукт

Рекомендовані формулювання:
- Telegram-first commerce platform
- платформа для запуску продажів через Telegram
- магазин у Telegram з керуванням через WebApp
- операційна система для Telegram-магазину

## Чого не треба обіцяти без перевірки

Не варто автоматично стверджувати:
- що всі модулі доступні на будь-якому тарифі
- що продукт повністю замінює будь-який e-commerce сайт у всіх сценаріях
- що всі інтеграції повністю автоматичні без ручних дій
- що платформа підтримує те, чого немає в `insomnia.workshop`

## Сенс лендингу

Лендинг має продавати не "бота", а повноцінну платформу для продажів через Telegram:
- запуск
- операційка
- оплати
- доставка
- контроль замовлень
- ріст через аналітику та кампанії
