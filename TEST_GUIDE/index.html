    🔐 Частина 1: Перевірка доступу

    1.1 Дізнатися свій Telegram ID

    Використайте будь-якого бота типу @userinfobot.

    1.2 Перевірка доступу адміна

    /admin

    Якщо ви в списку ADMIN_IDS: Відкриється адмін-меню Якщо ні: "Немає доступу до адмін-меню..."

    1.3 Edge Case: Неіснуюча команда

    /unknowncommand

    Очікувано: Бот ігнорує або показує меню

    👥 Частина 2: Управління адміністраторами

    2.1 Додати нового адміна

    1. /admin → ⚙️ Налаштування → 👥 Адміни
    2. Натисніть ➕ Додати адміна
    3. Введіть Telegram ID нового адміна (наприклад: 123456789)

    Очікувано: "✅ Адміністратора 123456789 додано!"

    2.2 Edge Cases для додавання адміна:

    • Текст замість числа: Введіть hello

    • Очікувано: "⚠️ Невірний формат. Введіть числовий Telegram ID."

    • Від'ємне число: Введіть -123

    • Очікувано: "⚠️ Невірний формат..."

    • Вже існуючий адмін: Введіть ID, який вже є

    • Очікувано: "ℹ️ Адміністратор ... вже є в списку."

    2.3 Видалити адміна

    1. 👥 Адміни → 🗑 Видалити {ID}

    Очікувано: "✅ Адміністратора ... видалено."

    2.4 Edge Case: Видалення останнього адміна

    Спробуйте видалити всіх адмінів по черзі.

    Очікувано: "⚠️ Не можна видалити останнього адміністратора!"

    📦 Частина 3: Управління товарами

    3.1 Створення товару

    1. /admin → 📦 Товари → ➕ Додати товар
    2. Введіть назву: Тестова футболка
    3. Введіть опис: Бавовна 100%
    4. Введіть ціну: 599
    5. Введіть розміри: S:5,M:10,L:3

    Очікувано: "Товар створено ✅"

    3.2 Edge Cases для створення товару:

    • Порожня назва: Просто Enter

    • Очікувано: "Назва не може бути порожньою"

    • Нечислова ціна: abc

    • Очікувано: "Ціна має бути числом > 0"

    • Від'ємна ціна: -100

    • Очікувано: "Ціна має бути числом > 0"

    • Невірний формат розмірів: S-5,M-10

    • Очікувано: "Не вдалося розпарсити. Спробуйте ще раз."

    3.3 Редагування товару

    1. 📦 Товари → Виберіть товар → ✏️ Назва
    2. Введіть нову назву

    3.4 Вимкнення/увімкнення товару

    1. У картці товару натисніть Вимкнути
    2. Перевірте каталог — товар не повинен відображатися
    3. Увімкніть назад

    3.5 Управління фото

    1. 🖼 Фото → ➕ Додати фото
    2. Надішліть фото
    3. Спробуйте додати 5+ фото

    Edge Case: Надіслати не фото (текст, стікер) - Очікувано: "Це не фото. Надішліть фото ще раз."

    3.6 Управління розмірами/складом

    1. 📐 Розміри / склад → Виберіть розмір
    2. Введіть новий stock: 20

    Edge Cases: - Від'ємний stock: -5 - Очікувано: "stock має бути числом >= 0"

    • Stock менше reserved: (якщо є бронювання)

    • Очікувано: "Не можна встановити stock=X, бо вже є reserved=Y"

    • Видалення розміру з reserved > 0:

    • Очікувано: "Не можна видалити розмір: є активні бронювання..."

    🛒 Частина 4: Покупки (як клієнт)

    4.1 Перегляд каталогу

    /catalog

    Очікувано: Список товарів з номерами [1] [2] ...

    4.2 Edge Case: Порожній каталог

    Вимкніть всі товари в адмін-панелі, потім /catalog

    Очікувано: "Наразі немає активних товарів."

    4.3 Перегляд товару

    Натисніть номер товару в каталозі

    Очікувано: Картка товару з фото та кнопками розмірів

    4.4 Каруселька фото

    Якщо у товару кілька фото, натискайте ◀️ / ▶️

    4.5 Додавання в кошик

    1. Натисніть розмір (напр. M)
    2. Перевірте повідомлення "Додано в кошик"

    4.6 Edge Case: Розмір out of stock

    Якщо stock = 0 для розміру, натисніть його

    Очікувано: "На жаль, цього розміру вже немає в наявності."

    4.7 Перегляд кошика

    /cart

    Очікувано: Список товарів у кошику з цінами

    4.8 Edge Case: Порожній кошик

    /cart

    без додавання товарів

    Очікувано: "Кошик порожній."

    🚚 Частина 5: Оформлення замовлення

    5.1 Happy Path

    1. /cart → Оформити замовлення
    2. Введіть місто: Київ
    3. Введіть відділення: 25
    4. Введіть ПІБ: Іван Петренко
    5. Введіть телефон: +380501234567

    Очікувано: Замовлення створено з посиланням на оплату

    5.2 Edge Cases для доставки:

    Порожнє місто: - Просто Enter - Очікувано: "Будь ласка, введіть місто (не порожньо)."

    Невірне відділення: - abc або -5 - Очікувано: "Невірний номер відділення..."

    Невірний ПІБ: - Іван (одне слово) - Очікувано: "Невірний формат ПІБ..."

    Невірний телефон: - 123 або телефон - Очікувано: "Невірний формат телефону..."

    5.3 Скасування оформлення

    Під час оформлення натисніть ❌ Скасувати

    Очікувано: "Оформлення доставки скасовано. Товари залишилися в кошику."

    5.4 Переривання оформлення

    Під час оформлення напишіть /catalog або /cart

    Очікувано: Доставка призупиняється, можна продовжити пізніше

    5.5 Відновлення оформлення

    Після переривання йдіть у /cart

    Очікувано: Кнопки "▶️ Продовжити оформлення" та "❌ Скасувати"

    5.6 Таймаут оформлення (15 хв)

    Почніть оформлення і зачекайте 15+ хвилин, потім продовжіть

    Очікувано: Стан скидається, потрібно почати знову

    💳 Частина 6: Оплата

    6.1 Повідомлення про оплату

    Після створення замовлення натисніть Я оплатив(ла)

    Очікувано: - Клієнт: "Дякуємо! Ми отримали повідомлення про оплату..." - Адміни: Повідомлення з деталями замовлення

    6.2 Перевірка статусу

    Натисніть Статус замовлення

    Очікувано: Інформація про статус (PendingPayment)

    6.3 Підтвердження оплати (адмін)

    1. /admin → 📋 Замовлення → 🕒 Очікують
    2. Виберіть замовлення → ✅ Mark Paid

    Очікувано: - "Оплату підтверджено ✅" - Клієнту: "Оплату підтверджено!" - Stock зменшується

    6.4 Edge Case: Повторне підтвердження

    Натисніть ✅ Mark Paid ще раз

    Очікувано: "Замовлення вже було Paid ✅"

    🧹 Частина 7: Cleanup та edge cases

    7.1 Cleanup старих замовлень

    📋 Замовлення → 🧹 Очистити (30 хв)

    Очікувано: Старі PendingPayment замовлення стають Expired, stock повертається

    7.2 Edge Case: Товар видалено після додавання в кошик

    1. Додайте товар у кошик
    2. Вимкніть товар в адмін-панелі
    3. Спробуйте оформити замовлення

    Очікувано: Помилка або пропуск неактивного товару

    7.3 Edge Case: Stock закінчився

    1. Встановіть stock = 1 для розміру
    2. Клієнт A додає в кошик
    3. Клієнт B намагається додати той самий розмір

    Очікувано для B: "На жаль, цього розміру вже немає в наявності."

    ⚙️ Частина 8: Налаштування магазину

    8.1 Редагування текстів

    ⚙️ Налаштування → 📝 Тексти магазину → Виберіть текст → Введіть новий

    Тест: Змініть "Привітання при /start", потім /start

    8.2 Редагування кнопок

    ⚙️ Налаштування → 🔘 Кнопки / назви кнопок

    8.3 Налаштування способів оплати

    💳 Оплата → увімкніть/вимкніть потрібні способи: - За посиланням - За реквізитами - Накладений платіж

    8.3.1 Зміна посилання для оплати

    💳 Оплата → ✏️ Змінити посилання

    8.3.2 Зміна реквізитів

    💳 Оплата → ✏️ Змінити реквізити

    8.4 Перегляд підписки

    📋 Підписка магазину

    Очікувано: Статус, дата оплати, план (read-only)

    🔄 Частина 9: Скидання стану

    9.1 Повний reset

    /reset

    Очікувано: "Стан скинуто ✅"

    📋 Чек-лист швидкого тесту

    • [ ] /start показує привітання
    • [ ] /admin відкриває меню (для адміна)
    • [ ] Додавання адміна працює
    • [ ] Видалення адміна працює (крім останнього)
    • [ ] Створення товару працює
    • [ ] Каталог показує товари
    • [ ] Додавання в кошик працює
    • [ ] Оформлення замовлення працює
    • [ ] "Я оплатив" надсилає сповіщення адмінам
    • [ ] Mark Paid змінює статус
    • [ ] Налаштування редагуються (/admin → ⚙️ Налаштування)
    • [ ] Перемикання методів оплати працює (посилання/реквізити/накладений)

    ⚠️ Відомі обмеження

    1. Підписка: Система ручна — автоматичних платежів немає
    2. Monobank webhook: Потребує E2E тесту з реальним платежем
    3. Відео/GIF: Поки не підтримуються в каруселі товарів
