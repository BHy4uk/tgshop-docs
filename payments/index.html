    Оплати¶

    Ця сторінка винесена окремо, щоб її можна було надсилати окремим посиланням. Зараз основний акцент — на підключенні Monobank Merchant, але надалі тут можна доповнювати інші способи оплати.

    Які способи оплати підтримуються зараз¶

    У поточній платформі магазин може працювати з такими способами оплати:

    • Monobank Merchant
    • Оплата за посиланням
    • Оплата за реквізитами
    • Накладений платіж

    У WebApp ці налаштування зібрані в секції Оплати в картці Оплати клієнтів.

    У боті той самий блок відкривається через:

    /admin → ⚙️ Налаштування → 💳 Оплати

    Як виглядають поля в інтерфейсі¶

    У WebApp¶

    У картці Оплати клієнтів власник бачить саме такі елементи:

    • Monobank Merchant увімкнено
    • Оплата за посиланням
    • Оплата за реквізитами
    • Післяплата
    • поле Посилання для оплати
    • поле Merchant Token Monobank
    • поле Текст реквізитів
    • кнопку Зберегти оплату

    У боті¶

    У боті в налаштуваннях оплат доступні такі дії:

    Що підтримується зараз¶

    • Monobank Merchant
    • Оплата за посиланням
    • Оплата за реквізитами
    • Накладений платіж

    У WebApp це секція Оплати, картка Оплати клієнтів.

    Де вставити токен у магазині¶

    У WebApp власник бачить такі поля:

    • Monobank Merchant увімкнено
    • Оплата за посиланням
    • Оплата за реквізитами
    • Післяплата
    • Посилання для оплати
    • Merchant Token Monobank
    • Текст реквізитів
    • Зберегти оплату

    Рекомендований флоу для Monobank Merchant¶

    1. Напишіть у Support Monobank Business, що вам потрібен інтернет-еквайринг для магазину / WebApp.
    2. Отримайте від support актуальний onboarding-флоу.
    3. Заповніть форму або заявку на еквайринг.
    4. Дочекайтеся активації мерчанта.
    5. Після активації зайдіть у https://web.monobank.ua/ і візьміть production token для merchant API.
    6. Вставте токен у поле Merchant Token Monobank у вашому магазині.
    7. Натисніть Зберегти оплату.
    8. Зробіть тестове замовлення.

    Чому саме так

    Публічна документація Monobank добре описує acquiring API, але не дає стабільного публічного click-path для бізнес-онбордингу мерчанта. Тому найнадійніший шлях — спочатку support, потім форма на еквайринг, потім production token.

    Приклад повідомлення в Support¶

    Вітаю. Потрібно підключити Monobank Merchant для інтернет-еквайрингу нашого магазину.

    Ми продаємо товари онлайн через Telegram WebApp / сайт-магазин і хочемо приймати оплати карткою через Monobank Merchant API.

    Підкажіть, будь ласка: 1. який актуальний порядок підключення інтернет-еквайрингу; 2. де заповнити заявку або форму на підключення; 3. які документи або дані від нас потрібні; 4. де після активації мерчанта взяти production Merchant Token / X-Token для API.

    Якщо є окрема інструкція або посилання на onboarding-форму, будь ласка, надішліть їх.

    Що підготувати до звернення¶

    • назву ФОП / компанії
    • ЄДРПОУ або ІПН ФОП
    • посилання на магазин або короткий опис checkout-сценарію
    • короткий опис товарів або послуг
    • контактну особу
    • email і телефон

    Де шукати Merchant Token Monobank¶

    • test token: https://api.monobank.ua/
    • production merchant token / X-Token: https://web.monobank.ua/ після активації інтернет-еквайрингу

    Важливо

    Публічна документація Monobank підтверджує, що production X-Token береться з web.monobank.ua, але не публікує стабільний маршрут усередині бізнес-кабінету. Якщо після активації мерчанта не бачите розділ із токеном, попросіть support дати точний поточний шлях у кабінеті.

    Як підключити токен у магазині¶

    Через WebApp¶

    1. Відкрийте Налаштування.
    2. Відкрийте секцію Оплати.
    3. Знайдіть картку Оплати клієнтів.
    4. Увімкніть Monobank Merchant увімкнено.
    5. Вставте токен у Merchant Token Monobank.
    6. Натисніть Зберегти оплату.

    Через бот¶

    1. Виконайте /admin.
    2. Відкрийте ⚙️ Налаштування.
    3. Натисніть 💳 Оплати.
    4. Увімкніть Monobank Merchant.
    5. Натисніть 🔐 Ввести Merchant Token.
    6. Надішліть токен окремим повідомленням.

    Як перевірити, що все працює¶

    1. Створіть тестове замовлення.
    2. Пройдіть checkout.
    3. Оплатіть через Monobank.
    4. Перевірте, що замовлення дійшло до ✅ Оплачено.

    Для тестового середовища Monobank дозволяє використовувати токен з https://api.monobank.ua/.

    Офіційні посилання Monobank¶

    • Acquiring API: https://api.monobank.ua/docs/acquiring.html
    • Авторизація для тестового токена: https://api.monobank.ua/
    • Open API docs: https://api.monobank.ua/docs/
    • Corporate API docs: https://api.monobank.ua/docs/corporate.html

    • Бізнес-кабінет: https://web.monobank.ua/ Для реального магазину найкраще діяти так:

    • Спочатку написати в Support Monobank Business і підтвердити, що вам потрібен саме інтернет-еквайринг для сайту / WebApp.

    • Після відповіді сапорту заповнити форму або пройти onboarding, який вони дадуть саме для вашого бізнесу.
    • Дочекатися активації мерчанта.
    • Після активації зайти у кабінет https://web.monobank.ua/ і отримати production token для merchant API.
    • Вставити цей токен у ваш магазин і зробити тестове замовлення.

    Чому саме так

    Публічна документація Monobank добре описує сам API еквайрингу, але не дає стабільного покрокового онбордингу для підключення мерчанта через бізнес-підтримку. Тому support-first флоу тут правильніший і практичніший, ніж намагатися вгадати порядок дій у кабінеті.

    Офіційні посилання Monobank, які варто мати під рукою¶

    • Acquiring API: https://api.monobank.ua/docs/acquiring.html
    • Вхід / авторизація для тестового токена: https://api.monobank.ua/
    • Open API docs: https://api.monobank.ua/docs/
    • Corporate API docs: https://api.monobank.ua/docs/corporate.html
    • Бізнес-кабінет Monobank: https://web.monobank.ua/

    Що говорить офіційна документація Monobank¶

    З офіційної acquiring-документації Monobank випливає таке:

    • для merchant API використовується X-Token
    • для production цей токен береться з особистого кабінету https://web.monobank.ua/
    • для тестового середовища можна використовувати токен з https://api.monobank.ua/
    • при створенні invoice можна передавати webHookUrl
    • на зміну статусу рахунку Monobank виконує до 3 спроб POST-вебхука, поки не отримає 200 OK

    Це прямо узгоджується з нашим сценарієм магазину, де Monobank Merchant потрібен для автоматичного підтвердження оплат.

    Якщо є окрема інструкція для бізнес-кабінету або onboarding-форма, будь ласка, надішліть посилання.

    Що підготувати до звернення в Support¶

    • назву ФОП / компанії

    Після відповіді сапорту: що робити далі¶

    Коли support надішле інструкцію або форму на еквайринг, дійте так:

    1. Заповніть форму на інтернет-еквайринг.
    2. Надішліть усі дані або документи, які запитає банк.
    3. Дочекайтеся підтвердження, що мерчант активовано.
    4. Після цього відкрийте https://web.monobank.ua/.
    5. Знайдіть токен для merchant/acquiring API.

    Важливе уточнення про шлях у кабінеті

    У публічній документації Monobank є підтвердження, що production X-Token береться з https://web.monobank.ua/, але немає стабільно задокументованого click-path усередині кабінету. Якщо після активації мерчанта ви не бачите розділ із токеном, не вгадуйте назви меню: просто попросіть support дати точний поточний маршрут у вашому бізнес-кабінеті.

    Де шукати Merchant Token Monobank¶

    Безпечне правило таке:

    • test token — через https://api.monobank.ua/
    • production merchant token / X-Token — у кабінеті https://web.monobank.ua/ після активації інтернет-еквайрингу

    Якщо вам потрібен саме production token для магазину, орієнтуйтеся не на api.monobank.ua, а саме на web.monobank.ua і на інструкцію від сапорту після активації мерчанта.

    Що потрібно до початку¶

    • активний бізнес-акаунт або еквайринг Monobank
    • доступ до Merchant Token у Monobank
    • доступ до адмінки магазину

    Варіант 1. Підключення через WebApp¶

    1. Відкрийте WebApp магазину.
    2. Перейдіть у вкладку Налаштування.
    3. Відкрийте секцію Оплати.
    4. Знайдіть картку Оплати клієнтів.
    5. Увімкніть перемикач Monobank Merchant увімкнено.
    6. Вставте токен у поле Merchant Token Monobank.
    7. Натисніть Зберегти оплату.

    Що відбувається після збереження

    Токен зберігається на бекенді в зашифрованому вигляді. Після цього обов’язково зробіть тестове замовлення і перевірте весь сценарій до кінця.

    Варіант 2. Підключення через бот¶

    1. Виконайте /admin.
    2. Відкрийте ⚙️ Налаштування.
    3. Натисніть 💳 Оплати.
    4. Увімкніть Monobank Merchant.
    5. Натисніть 🔐 Ввести Merchant Token.
    6. Надішліть Merchant Token наступним повідомленням.

    Як поводиться бот

    У боті токен не показується назад у відкритому вигляді. Система просить надіслати його окремим повідомленням і далі зберігає без відображення в адмінці.

    Як протестувати Monobank Merchant¶

    Після підключення перевірте оплату на реальному сценарії:

    1. Відкрийте магазин як покупець.
    2. Додайте товар у кошик.
    3. Пройдіть checkout.
    4. Оберіть Monobank-оплату.
    5. Завершіть платіж.
    6. Перевірте, що замовлення потрапило в правильний статус.

    Як протестувати без production-доступу¶

    Офіційна acquiring-документація Monobank окремо зазначає, що тестове середовище доступне всім клієнтам банку через токен з https://api.monobank.ua/. Це корисно, якщо ви хочете спочатку перевірити саму інтеграцію, вебхуки і логіку зміни статусів ще до фінального production-підключення.

    Які назви побачить покупець¶

    У клієнтському сценарії для Monobank можуть використовуватись такі кнопки:

    • 💳 Оплатити онлайн (Monobank)
    • 💳 Оплатити
    • Я оплатив(ла)
    • Статус замовлення

    Який статус очікувати¶

    Для успішної онлайн-оплати дивіться на такі статуси:

    • ⏳ Очікує оплати — замовлення створене, оплата ще не підтверджена
    • 🧾 Оплата на перевірці — клієнт натиснув Я оплатив(ла), а ви або система ще не завершили підтвердження
    • ✅ Оплачено — цільовий стан для успішно оплаченої онлайн-оплати

    Не вважайте інтеграцію готовою без тесту

    Сам факт, що токен збережено, ще не означає, що платіжний сценарій перевірений. Готовність підтверджує тільки тестове замовлення, яке дійшло до ✅ Оплачено.

    Інші способи оплати¶

    Оплата за посиланням¶

    Підходить, якщо ви хочете дати покупцю зовнішнє посилання на оплату, наприклад Monobank jar.

    У WebApp для цього є:

    • перемикач Оплата за посиланням
    • поле Посилання для оплати

    У боті є дії:

    • За посиланням — Увімкнути / Вимкнути
    • ✏️ Змінити посилання

    Для покупця цей спосіб відображається як:

    • 💳 Онлайн оплата
    • або в WebApp як Оплата за посиланням

    Оплата за реквізитами¶

    Підходить, якщо покупець має вручну переказати кошти на ваші реквізити.

    У WebApp для цього є:

    • перемикач Оплата за реквізитами
    • поле Текст реквізитів

    У боті є дії:

    • За реквізитами — Увімкнути / Вимкнути
    • ✏️ Змінити реквізити

    Для покупця цей спосіб відображається як:

    • 🏦 Банківські реквізити
    • або в WebApp як Оплата за реквізитами

    Після переказу покупець натискає Я оплатив(ла), а замовлення переходить у 🧾 Оплата на перевірці.

    Накладений платіж¶

    У WebApp у власника цей спосіб вмикається перемикачем Післяплата, а в боті — Накладений платіж — Увімкнути / Вимкнути.

    Для покупця цей спосіб відображається як:

    • 📦 Накладений платіж

    Після цього замовлення йде по COD-логіці:

    • 📦 Очікує відправки
    • 🚚 Відправлено
    • 📍 Очікує отримання
    • ✅ Оплачено

    Важливе правило для товарів під замовлення¶

    Якщо у вас є товари з режимом untracked (під замовлення), їм потрібна хоча б одна онлайн-оплата:

    • Monobank Merchant
    • Оплата за посиланням
    • Оплата за реквізитами

    Якщо залишити тільки накладений платіж, checkout для таких товарів буде заблокований.
